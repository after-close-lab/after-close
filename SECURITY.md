# 安全与隐私

## 私密报告

如果发现密钥泄漏、账户信息、身份信息或其他不应公开的内容，请不要创建公开 Issue。请使用 GitHub 仓库的 Private vulnerability reporting 功能联系 Maintainer。

## 禁止提交

- 券商、银行或其他金融账户号码；
- 订单号、身份证件、电话号码和家庭地址；
- API Key、Personal Access Token、Cookie 或私钥；
- 完整账户截图、对账单或可推算完整资产规模的组合信息；
- 未经授权的他人交易数据。

## 泄漏处理

1. 立即吊销或轮换相关凭证；
2. 暂停 Pages 部署与受影响的自动化；
3. 由 Maintainer 评估是否需要 Git 历史重写；
4. 通知可能受影响的成员；
5. 修复扫描规则并记录事件原因；
6. 在确认风险解除后恢复发布。

仅删除当前分支中的文件不能从 Git 历史中移除敏感信息。
